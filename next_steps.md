## ⭐ NEXT BUILD PRIORITY — Templates redesign (DESIGN DONE 2026-06-20, ADR-093; build next session)

**The next build chunk.** A design session with Brandon settled the shape; **no code yet.** Recorded as **ADR-093** (CORE — reverses ADR-045's "templates aren't items"; **flag Tyler in `COLLAB.md` to ratify before the TPL1 schema migration merges**). **The four decisions (Brandon, 2026-06-20):**
1. **Templates are real items.** A template's content is a normal `items` row + subtree, authored/edited in the same canvas as any item (real subtasks/body/properties/relations — no second editor). Apply = a deep clone. (Reverses ADR-045; **CORE**.)
2. **Variables = dates + ask-on-apply prompts + computed (full tier).** `{{today}}`/`{{today+7d}}`/`{{now}}`/`{{nextSunday}}`/`{{title}}` tokens in title+body, each fixed-or-dynamic; relative due/scheduled offsets; `{{ask:Label}}` fill-in prompts collected on apply.
3. **Apply-to-existing = fill-blanks (default) + overwrite (explicit).** Apply to an already-started item, filling only empty fields + adding missing subtasks (Brandon's "change only the unchanged"); plus an explicit overwrite mode.
4. **Create UX = per-type default + chooser.** Each type may set a default template "+ New" uses; a chooser lists the rest + Blank.

**Industry grounding (research, 2026-06-20):** Notion (duplicate-a-prototype + `@today` "date when duplicated" toggle + default DB template), ClickUp (merge-on-apply-to-existing, per-list default, remap-subtask-dates, granular "what to import"), Obsidian Templater (`tp.date.now(fmt, offset)` + `tp.system.prompt`), Asana (subtasks + relative dates; notably *can't* do relative-to-parent-due, which Ledgr's S5 relative subtasks already nails).

## ⟢ NOW BUILDING — Mobile kanban drag-and-drop (long-press to drag on touch, 2026-06-19)

Brandon: kanban DnD works on desktop but not mobile. Add long-press-and-drag on touch. The board (`BoardDnd.tsx`, ADR-072) uses native HTML5 drag, which is **mouse-only** — touch never fires `dragstart`/`drop`, and mobile browsers have no native drag for arbitrary elements. Fix = add an isolated **touch** input path beside the untouched desktop mouse path (no DnD library — Principle 5, matches the board's existing "no DnD dependency" stance). The drop logic (`drop(col)`/`moveCard`/`boardDropPatch`, optimistic + revert + `router.refresh()`, `dragId`/`overCol`) is fully reused — **no API/schema/persistence change.** Non-core (one view surface, no dependency added → no ADR); solo.

**Gesture:** `touchstart` → ~400ms long-press timer; finger moves >8px before it fires → scroll/tap, cancel (browser scrolls); timer fires (held still) → arm (haptic + lift), `preventDefault` from here; `touchmove` → hit-test column under finger (`elementFromPoint` → `[data-col]`), set `overCol`; `touchend` → `drop(overCol)` + suppress the click so the `<Link>` doesn't navigate; `touchcancel` → revert. **Key gotcha:** React's `onTouchMove` is passive, so the non-passive `touchmove` listener is attached imperatively via the container ref (`{passive:false}`), and preventDefault fires only while armed. Also tags the board `data-scroll-x` (the `mobile-swipe-navigation.md` conflict marker).

- [x] **K1 — Touch long-press drag. ✅ DONE (2026-06-19).** Pure helpers `src/lib/board-touch-drag.ts` (`exceedsMoveThreshold` scroll-vs-hold at 8px, `columnAtPoint` direct-hit + nearest-column fallback, `LONG_PRESS_MS`/`MOVE_CANCEL_PX`); the `useBoardTouchDrag(boardRef, callbacks)` hook (`src/components/views/useBoardTouchDrag.ts`) — delegated listeners on the container, **non-passive `touchmove`** attached imperatively (React's is passive, can't preventDefault), ~400ms long-press arm + `navigator.vibrate`, per-move rect-based column hit-test, click-suppression via touchend `preventDefault`, latest-callbacks ref so mid-drag re-renders don't stale the listeners. - **Bugfix — stranded @-mention "No matches" popup (2026-06-14, Brandon first-use):** the hand-rolled mention popup (`src/components/markdown-editor/mention-suggestion.ts`) is a detached `document.body` node whose only teardown paths were Tiptap's `onExit` and `Escape`. **Culprit:** `onExit` fires only when the matched range changes in the doc, so typing `@` and then clicking away (a blur, no doc change) never closed it; and the node lives outside React, so a route change unmounted the editor without removing it — it survived until a full refresh.
